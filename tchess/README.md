# T-CHESS — Telegram-бот с мини-приложением

Шахматный бот с онлайн-партиями 1×1, тактическими задачами, турнирной статистикой
и профилем игрока (T-CHESS ID). Спонсоры **TikhonCHESS** и **Блог Шахматиста**
показаны баннером на главной и подписью внизу страниц.

## ⚠️ Сначала — про токен

Вы присылали токен бота в переписке. Как только всё заработает, зайдите к
[@BotFather](https://t.me/BotFather) → `/mybots` → выберите бота → **API Token** → **Revoke**,
чтобы выпустить новый токен. Старый нигде больше не используйте. Новый токен храните
только в файле `.env` (он уже добавлен в `.gitignore` и никуда не публикуется).

## Что понадобится

- Аккаунт в Telegram и доступ к [@BotFather](https://t.me/BotFather)
- Аккаунт на [Render.com](https://render.com) (бесплатно, карта не нужна)
- Аккаунт на [GitHub](https://github.com) — Render разворачивает сервис из git-репозитория

## Деплой на Render (рекомендуется)

Render даёт настоящий облачный сервер с постоянным HTTPS-адресом, который никогда
не меняется, и работает независимо от вашего компьютера — его можно выключить.
Единственная особенность бесплатного тарифа — сервис "засыпает" после ~15 минут
без запросов, первое обращение после сна грузится ~30–50 секунд. Подробнее — в
разделе [«Про бесплатный тариф Render»](#про-бесплатный-тариф-render) ниже.

**1. Залейте проект в GitHub**

Если ещё не сделано — создайте пустой репозиторий на GitHub и загрузите туда
папку `tchess` (через `git push` или прямо через сайт GitHub, кнопка "Add file →
Upload files"). Файл `.env` заливать не нужно и не надо — он и так в `.gitignore`.

**2. Создайте сервис на Render**

1. Зайдите на [render.com](https://render.com) → **New** → **Blueprint**
2. Подключите ваш GitHub-репозиторий с проектом — Render сам найдёт файл
   `render.yaml` и предложит настройки (они уже готовы в проекте)
3. Render попросит ввести переменную `BOT_TOKEN` — вставьте токен от @BotFather
4. Нажмите **Apply** / **Create** — начнётся первый деплой (2–3 минуты)

Если предпочитаете без Blueprint — **New → Web Service**, выберите репозиторий,
Build Command `npm install`, Start Command `npm start`, Instance Type **Free**,
и вручную добавьте переменную окружения `BOT_TOKEN`.

**3. Готово**

После деплоя Render покажет постоянный адрес вида `https://tchess-xxxx.onrender.com`.
Сервер сам подставляет его в переменную `MINI_APP_URL` (через `RENDER_EXTERNAL_URL`,
это делает Render автоматически) и сам подключает вебхук Telegram — ничего
руками прописывать не нужно. Откройте бота в Telegram и отправьте `/start`.

## Настройка кнопки мини-приложения в BotFather

1. `/mybots` → выберите бота → **Bot Settings** → **Menu Button** → **Configure Menu Button**
2. Отправьте адрес вашего сервиса на Render, например `https://tchess-xxxx.onrender.com`
3. Название кнопки, например: `♟ T-CHESS`

Теперь у бота внизу чата будет постоянная кнопка запуска мини-приложения, а команда
`/start` дополнительно пришлёт баннер со ссылкой на TikhonCHESS и Блог Шахматиста.
Адрес на Render не меняется при перезапусках/пробуждении — прописать его в BotFather
нужно один раз.

## Про бесплатный тариф Render

- **Сон после простоя.** Без запросов ~15 минут сервис засыпает. Бот работает через
  вебхук (а не long polling), поэтому даже "спящий" сервис проснётся от входящего
  сообщения в Telegram — просто первый ответ придёт с задержкой 30–50 секунд, дальше
  всё быстро, пока идёт активность.
- **Если хочется, чтобы бот совсем не засыпал** (например, для удобства во время игры
  с другом) — заведите бесплатный аккаунт на [UptimeRobot](https://uptimerobot.com)
  и добавьте HTTP-монитор на ваш адрес `https://tchess-xxxx.onrender.com` с
  интервалом проверки 5–10 минут. Регулярные запросы не дадут сервису уснуть.
  Это не обязательно, просто удобная опция.
- **Хранилище данных (рейтинг, история партий).** По умолчанию, если не задать
  `DATABASE_URL`, бот хранит всё в JSON-файлах в `data/` — а файловая система
  бесплатного тарифа Render эфемерная: эти файлы стираются при каждом передеплое
  и после долгого простоя (сна). Для дружеского теста это не страшно — рейтинг
  просто обнулится, сама игра продолжит работать как обычно. Если хочется, чтобы
  рейтинг и история не терялись — подключите бесплатную внешнюю базу Postgres
  (см. раздел [«Постоянное хранилище (Postgres)»](#постоянное-хранилище-postgres-опционально)
  ниже), это уже готово в коде, нужно только вписать один адрес подключения.

## Постоянное хранилище (Postgres, опционально)

По умолчанию рейтинг и история партий хранятся в JSON-файлах и обнуляются при
передеплое (см. выше). Чтобы данные сохранялись постоянно, подключите бесплатную
базу — например, [Neon](https://neon.tech) (2–3 минуты, карта не нужна):

1. Зарегистрируйтесь на [neon.tech](https://neon.tech) и создайте проект
   (бесплатного тарифа хватает с большим запасом для теста и небольшой группы игроков)
2. В панели проекта скопируйте **Connection string** — строку вида
   `postgresql://user:password@ep-xxxx.neon.tech/neondb?sslmode=require`
3. На Render: **Dashboard → ваш сервис → Environment** → добавьте переменную
   `DATABASE_URL` со скопированной строкой → **Save Changes** (сервис сам
   передеплоится с новой настройкой)
4. Для локального запуска — впишите ту же строку в `DATABASE_URL` в файле `.env`

При первом запуске сервер сам создаст нужные таблицы в базе — руками ничего
настраивать не нужно. Если `DATABASE_URL` не задан, всё продолжает работать как
раньше, на JSON-файлах — эту переменную можно оставить пустой и подключить базу
в любой момент позже, старые локальные данные из `data/` при этом не переносятся
автоматически (для дружеского теста это не важно — можно просто начать заново).

Подходит и [Supabase](https://supabase.com) — там тоже есть бесплатный Postgres
со строкой подключения в настройках проекта (Project Settings → Database →
Connection string, режим "URI").

## Локальный запуск (для разработки)

```bash
cd tchess
npm install
cp .env.example .env
```

Впишите в `.env` только `BOT_TOKEN` — без публичного адреса сервер сам включит
режим long polling, подходящий для локальной разработки:

```bash
npm start
```

Если нужно открыть мини-приложение локально в Telegram (не только тестировать API),
понадобится свой HTTPS-туннель — впишите его адрес в `MINI_APP_URL` в `.env`:
```
cloudflared tunnel --url http://localhost:3000    # или: ngrok http 3000
```

## Как это устроено

```
tchess/
├── server.js         # Express + WebSocket + Telegram-бот (вебхук на Render, long polling локально)
├── storage.js         # Хранилище: Postgres, если задан DATABASE_URL, иначе JSON-файлы
├── data/              # JSON-хранилище (fallback-режим без базы, создаётся автоматически)
└── public/
    ├── index.html      # разметка мини-приложения (3 страницы: Главная / Турниры / Профиль)
    ├── style.css        # оформление в стиле шахматного клуба
    ├── app.js            # вся логика: доска, задачи, матчмейкинг, профиль
    └── puzzles.js         # база шахматных задач (можно свободно дополнять)
```

**Онлайн-партии.** Игрок нажимает «Найти соперника» → сервер ставит его в очередь
и сразу же соединяет с первым подходящим соперником через WebSocket → оба клиента
проверяют ходы локально (библиотека `chess.js`), сервер их только ретранслирует и
считает изменение рейтинга (формула Эло) по завершении партии.

**Задачи.** Список позиций (FEN) и решений лежит в `public/puzzles.js` — добавляйте
свои в том же формате, ничего в коде менять не нужно.

**Профиль / T-CHESS ID.** Внутри Telegram имя и фото подтягиваются автоматически
из Telegram (через `initData`, который сервер проверяет по официальной схеме
Telegram — `verifyInitData` в `server.js`). Кнопка «Войти» синхронизирует рейтинг
и историю партий с сервером; «Выйти» переключает в гостевой режим, где данные
хранятся только локально в этом устройстве.

## Известные упрощения (MVP)

- Хранилище — JSON-файлы по умолчанию, с готовым переключением на Postgres
  через `DATABASE_URL` (см. раздел выше). Для совсем большой аудитории стоит
  добавить пул соединений/индексы под нагрузку — для дружеских тестов и
  небольшой группы игроков текущей схемы достаточно.
- Часы партии считаются на каждом клиенте отдельно (без строгой серверной синхронизации) —
  для дружеских онлайн-партий этого достаточно, но не подходит для турниров с призами.
- Список задач — стартовый набор из 5 позиций, легко расширяется.
- Если два человека одновременно нажмут «Найти соперника» из очень разных часовых
  поясов/рейтингов, бот всё равно их сведёт — сортировки по рейтингу для матчмейкинга нет.

## Добавление рекламных баннеров

Слайды баннера — в `public/index.html`, блок `.sponsor-banner`. Добавьте ещё один
`<div class="sponsor-slide">…</div>` по образцу существующих — карусель подхватит его
автоматически (см. `rotateSponsorBanner()` в `app.js`).
